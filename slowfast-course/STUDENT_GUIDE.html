A

Qu'est-ce que PySlowFast ?

Comprendre ce que vous allez installer — en langage simple

Imaginez que vous regardez une vidéo de quelqu'un qui joue au basket. Pour comprendre ce qui se passe, vous devez faire deux choses en même temps :

  1. Percevoir la scène globale — À quoi ressemble le terrain ? Où sont les joueurs ?
  2. Capter les mouvements rapides — Le ballon se déplace-t-il ? Quelqu'un saute-t-il en ce moment ?

PySlowFast est un programme d'IA qui fait les deux simultanément, grâce à deux « voies » distinctes :

NomCe qu'elle faitÉquivalent humain
🐢 Voie Lente (Slow)Analyse peu d'images par seconde — perçoit clairement la scène globaleRegarder une photo
🐇 Voie Rapide (Fast)Analyse beaucoup d'images par seconde — capte les mouvements brusquesRegarder en temps réel

Résultat : une IA capable de regarder n'importe quelle vidéo et de dire « Cette personne joue de la guitare » ou « Cette vidéo montre quelqu'un qui cuisine ».

🎯
Ce que vous allez accomplir aujourd'hui Votre ordinateur va regarder une vidéo et identifier l'action qui s'y déroule, grâce à une IA entraînée sur 400 types d'actions humaines. Cette même technologie est utilisée dans les caméras de surveillance, l'analyse sportive et la surveillance médicale.

Quel type d'ordinateur ai-je ?

Identifiez votre configuration avant de commencer

Type d'ordinateurComment reconnaître le vôtreVitesse IACe qu'on installe
🪟 Windows + GPU NVIDIA PC portable ou fixe Windows avec une carte NVIDIA RTX, GTX ou similaire ⚡⚡⚡ Rapide PyTorch + CUDA 12.1 (accélération GPU complète)
🍎 Mac — Apple Silicon Mac acheté fin 2020 ou après, avec une puce M1, M2 ou M3.
Vérifier : Menu Apple → À propos de ce Mac → chercher « M1 » / « M2 » / « M3 »		 ⚡⚡ Modéré (CPU) PyTorch standard — MPS disponible mais on utilise le CPU pour la stabilité
🍎 Mac — Intel Ancien Mac avec processeur Intel Core i5/i7/i9.
Vérifier : À propos de ce Mac → chercher « Intel Core »		 ⚡ Lent (CPU uniquement) PyTorch CPU — aucune accélération GPU
💡
Qu'est-ce qu'un GPU ? Un GPU (processeur graphique) est une puce spéciale capable d'effectuer des milliers de calculs mathématiques simultanément. L'entraînement d'une IA nécessite des millions de calculs — un GPU les effectue 100 fois plus vite qu'un processeur classique. Les GPU NVIDIA utilisent le système CUDA sur Windows. Les Mac Apple Silicon disposent d'un moteur neuronal intégré, mais PySlowFast ne le supporte pas encore complètement — on utilise donc le mode CPU sur tous les Mac.
⚠️
Utilisateurs Mac — à savoir sur la vitesse PySlowFast a été conçu pour Linux/Windows avec des GPU NVIDIA. Sur Mac, il fonctionne en mode CPU uniquement, ce qui est 10 à 30 fois plus lent. Pour cette démo de cours avec de petites vidéos fictives, l'entraînement dure 5 à 30 minutes — acceptable pour apprendre. L'entraînement complet sur Kinetics-400 prendrait plusieurs jours sur Mac et n'est pas recommandé.

Installer PyTorch (le moteur IA)

L'installation la plus importante — le moteur qui fait tourner l'IA

🧠
Qu'est-ce que PyTorch ? PyTorch, c'est comme le moteur d'une voiture — PySlowFast est la carrosserie. Sans PyTorch, rien ne fonctionne. On installe une version spécialement adaptée à votre matériel pour que l'IA tourne aussi vite que possible sur votre machine.
🪟 Windows Installation avec CUDA 12.1 (pour RTX 3060)
ÉTAPE 3.1 — Installer PyTorch + CUDA
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
⏳
Ce téléchargement fait environ 2,5 Go. Comptez 5 à 15 minutes selon votre connexion. Ne fermez pas la fenêtre.
ÉTAPE 3.2 — Vérifier que le GPU fonctionne
python -c "import torch; print('PyTorch:', torch.__version__); print('GPU prêt:', torch.cuda.is_available()); print('Nom GPU:', torch.cuda.get_device_name(0))"
✅ Résultat attendu :
PyTorch: 2.x.x+cu121 GPU prêt: True Nom GPU: NVIDIA GeForce RTX 3060 Laptop GPU
🚫
Si GPU prêt: False — votre pilote NVIDIA doit peut-être être mis à jour. Rendez-vous sur nvidia.com/drivers et installez le dernier pilote pour votre GPU.
🍎 Mac Apple Silicon PyTorch avec support MPS
ÉTAPE 3.1-APPLE — Installer PyTorch

Le PyTorch moderne inclut MPS (le système GPU d'Apple) automatiquement. Installez simplement la version standard :

pip install torch torchvision torchaudio
⏳
Téléchargement d'environ 700 Mo. Comptez 2 à 5 minutes.
ÉTAPE 3.2-APPLE — Vérifier l'installation
python3 -c "import torch; print('PyTorch:', torch.__version__); print('MPS disponible:', torch.backends.mps.is_available())"
✅ Résultat attendu sur Mac M1/M2/M3 :
PyTorch: 2.x.x MPS disponible: True
ℹ️
On utilisera quand même le mode CPU pour PySlowFast
Même si MPS est disponible, PySlowFast présente des problèmes de compatibilité avec le backend MPS qui peuvent provoquer des plantages. On utilisera NUM_GPUS 0 pour forcer le mode CPU — plus lent, mais toujours stable.
🍎 Mac Intel Installation CPU uniquement
ÉTAPE 3.1-INTEL — Installer PyTorch
pip install torch torchvision torchaudio
ÉTAPE 3.2-INTEL — Vérifier l'installation
python3 -c "import torch; print('PyTorch:', torch.__version__); print('MPS:', torch.backends.mps.is_available())"
✅ Résultat attendu sur Mac Intel (MPS = False est normal) :
PyTorch: 2.x.x MPS: False
⚠️
Vitesse sur Mac Intel : L'entraînement de 5 époques sur le jeu de données fictif prend 15 à 40 minutes. Pour la démo de cours, utilisez le modèle pré-entraîné (Section 9) qui ne prend que quelques minutes même sur un CPU Intel.

Entraîner votre premier modèle

Regarder l'IA apprendre de zéro — c'est là que la magie opère

🤖
Qu'est-ce que l'entraînement ? L'IA examine des vidéos étiquetées et ajuste ses paramètres internes jusqu'à ce qu'elle puisse prédire correctement les étiquettes. Chaque passage complet sur toutes les vidéos d'entraînement s'appelle une époque. On lance seulement 5 époques pour que ça reste rapide.
ÉTAPE 7.1 — Entrer dans le dossier PySlowFast
🪟 Windows 
cd D:\CollegeCDI\machineLearning\slowfast
🍎 Mac 
cd ~/Documents/machineLearning/slowfast
ÉTAPE 7.2 — Lancer l'entraînement
✅
Tous les paramètres sont déjà dans le fichier YAML — si vous utilisez le dépôt du professeur, la commande est courte. Pas besoin de la longue liste d'options.
🪟 Windows PowerShell

Étape obligatoire — à répéter à chaque nouvelle session de terminal (évite un plantage lié au réseau Windows) :

$env:TORCH_DISTRIBUTED_USE_LIBUV = "0"

Puis lancer l'entraînement :

python tools/run_net.py --cfg configs/Kinetics/C2D_8x8_R50.yaml
ℹ️
Le fichier configs/Kinetics/C2D_8x8_R50.yaml contient déjà : dataset path, NUM_CLASSES 5, NUM_GPUS 1, BATCH_SIZE 4, MAX_EPOCH 5, OUTPUT_DIR, décodeur pyav, etc.
🍎 Mac — Apple Silicon & Intel (mode CPU)

Sur Mac, quelques paramètres doivent être passés en ligne de commande pour forcer le mode CPU et les bons chemins :

python3 tools/run_net.py \
  --cfg configs/Kinetics/C2D_8x8_R50.yaml \
  NUM_GPUS 0 \
  DATA.PATH_TO_DATA_DIR /tmp/synthetic_kinetics \
  OUTPUT_DIR /tmp/sf_output/c2d_test \
  DATA_LOADER.NUM_WORKERS 0
ℹ️
Conseil pour Mac Intel : Si l'entraînement est trop long, réduisez SOLVER.MAX_EPOCH à 2 et TRAIN.BATCH_SIZE à 1. Cela suffit à prouver que la chaîne de traitement fonctionne correctement.
⏳
Durées estimées pour 5 époques :
🪟 Windows RTX 3060 → environ 3 à 10 minutes
🍎 Mac Apple Silicon (M1/M2) → environ 10 à 20 minutes (mode CPU)
🍎 Mac Intel → environ 20 à 45 minutes

Glossaire

Chaque terme technique de ce guide expliqué en langage simple

TermeDéfinition en langage simple
IA / Intelligence ArtificielleUn programme qui apprend des motifs à partir d'exemples au lieu de suivre des règles codées manuellement
ModèleLe « cerveau » de l'IA — un fichier contenant des millions de nombres ajustés pendant l'entraînement
EntraînementMontrer à l'IA des milliers d'exemples étiquetés pour qu'elle apprenne à reconnaître des motifs
ÉpoqueUn passage complet sur toutes les données d'entraînement. Plus d'époques = plus d'apprentissage (jusqu'à un certain point)
Perte (Loss)Un chiffre mesurant à quel point l'IA se trompe. Plus c'est bas = mieux c'est. Doit diminuer pendant l'entraînement
Précision (Top-1)Le pourcentage de vidéos où la première prédiction de l'IA est l'étiquette correcte
GPUUne puce conçue pour effectuer des milliers de calculs simultanément — rend l'IA 100 fois plus rapide que le CPU
CUDALe système NVIDIA pour programmer les GPU depuis Python. Uniquement disponible sur le matériel NVIDIA (Windows)
MPSMetal Performance Shaders — le système d'accélération GPU d'Apple pour les Mac M1/M2/M3. Pas encore entièrement supporté par PySlowFast
Mode CPU uniquementFaire tourner l'IA sur le processeur principal plutôt que sur le GPU. Toujours fonctionnel mais plus lent
PyTorchLa bibliothèque d'IA la plus populaire. PySlowFast est construite par-dessus
Environnement virtuelUn dossier Python isolé. Garde les bibliothèques du projet séparées pour éviter les conflits
pipLe gestionnaire de paquets Python — l'outil utilisé pour installer des bibliothèques comme PyTorch
Point de contrôle (Checkpoint)Une copie sauvegardée de l'état du modèle. Permet de reprendre l'entraînement ou de partager l'IA
Config / YAMLUn fichier texte avec tous les paramètres qui contrôlent l'entraînement — comme une recette pour l'IA
Modèle pré-entraînéUn modèle que quelqu'un d'autre a déjà passé des jours à entraîner — vous le téléchargez et l'utilisez en quelques secondes
Kinetics-400Un jeu de données de 240 000 vidéos YouTube, chacune étiquetée avec l'un des 400 types d'actions humaines
Inférence / DémoUtiliser un modèle entraîné pour prédire sur de nouvelles vidéos (pas d'entraînement — juste l'utilisation du cerveau déjà formé)
TensorBoardUn tableau de bord dans le navigateur à localhost:6006 affichant des graphiques en direct de la progression de l'entraînement
Voie Lente / Voie RapideLes deux flux dans SlowFast : la voie Lente capte l'apparence de la scène, la voie Rapide capte les détails de mouvement
PYTHONPATHUne variable système indiquant à Python où chercher des paquets de code supplémentaires sur votre disque

🚀 Défis — Pour aller plus loin

DÉFI 1 — Facile (toutes les plateformes)

Tester l'IA sur différents types de vidéos

Trouvez 5 vidéos montrant des activités différentes : sport, cuisine, musique, exercice, danse. Exécutez chacune dans la démo. Notez quelles actions l'IA reconnaît correctement et lesquelles elle rate. Pouvez-vous expliquer pourquoi elle se trompe sur certaines vidéos ?

DÉFI 2 — Intermédiaire (Windows recommandé, Mac possible)

Entraîner sur vos propres vidéos

Filmez-vous en train de faire 3 actions différentes (saluer, applaudir, sauter — 20 vidéos chacune). Créez les fichiers CSV avec le script make_kinetics_csv.py et entraînez un modèle sur votre jeu de données personnel. Quelle précision obtenez-vous après 10 époques ?

DÉFI 3 — Difficile (Windows recommandé)

Comparer deux architectures de modèles

Entraînez C2D et X3D-M sur le même jeu de données fictif. Utilisez le script benchmark_models.py pour comparer leur vitesse et leur nombre de paramètres. Rédigez un court rapport : quel modèle déploieriez-vous dans une vraie application et pourquoi ?
